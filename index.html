How Super’s computer-use cache works

1. Real computer use

Agents click, type, scroll, authenticate, and navigate like a human—no brittle API-only assumptions.

2. Action trace capture

Successful steps are recorded as reusable traces, including UI landmarks and decisions.

3. Cache reuse

On the next run, the agent replays known-good paths instead of rediscovering them.

4. Guardrails

User confirmation gates sensitive actions; failures fall back to fresh exploration.
